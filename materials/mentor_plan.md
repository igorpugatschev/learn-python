# Менторский план по книге

Источник: `codelibs.ru_legkij-sposob-vyuchit-python-3-eshe-glubzhe.pdf`

## Как будем заниматься

1. Берем одно упражнение или небольшой блок.
2. Я кратко объясняю идею человеческим языком.
3. Вы пишете код в рабочей папке.
4. Я проверяю код, задаю уточняющие вопросы и показываю, что улучшить.
5. Фиксируем итог: что поняли, какие ошибки были, что повторить.

## Маршрут

### Этап 0. Подготовка

- Упражнение 0: настройка редактора, Python, терминала, pip/virtualenv, GitHub и git.
- Цель: сделать рабочую среду, где удобно писать, запускать и сохранять код.

### Этап 1. Учимся учиться

- Упражнение 1: движение.
- Упражнение 2: креативность.
- Упражнение 3: качество.
- Цель: выработать ритм практики, не просто читать книгу.

### Этап 2. Быстрые задания и командная строка

- Упражнения 4-12: аргументы командной строки, `cat`, `find`, `grep`, `cut`, `sed`, `sort`, `uniq`, обзор.
- Цель: научиться уверенно работать с файлами, текстом и терминалом.

### Этап 3. Структуры данных

- Упражнения 13-24: списки, стеки, очереди, сортировки, словарь, производительность, деревья, поиск, массивы.
- Цель: понимать не только синтаксис Python, но и устройство данных.

### Этап 4. Проекты следующего уровня

- Упражнения 25+ по книге.
- Цель: перейти от отдельных упражнений к проектному мышлению.

## Важная пометка по PDF

Обычное извлечение текста работает хорошо для чтения глав и оглавления. В примерах кода встречаются ошибки распознавания: `.py` может выглядеть как `.ру`, имена методов и кавычки иногда искажаются. Поэтому код из книги нужно проверять перед запуском.

MinerU может пригодиться как следующий шаг, если понадобится более чистый Markdown со структурой, таблицами и изображениями. Для старта менторства текущего извлечения достаточно.

## Первый урок

Начинаем с упражнения 0: проверяем среду.

Минимальный чек-лист:

- Python запускается из терминала.
- Можно создать и запустить файл `hello.py`.
- Есть папка для упражнений.
- Git установлен и показывает версию.
- Понятно, как запускать команды из книги в PowerShell.
