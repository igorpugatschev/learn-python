# Дизайн учебного notebook

## Контекст

Пилот аудиоверсии упражнения 4 показал, что локальный голос Windows не подходит по качеству. Аудионаправление отменено, а сгенерированные звуковые файлы удалены.

При этом адаптированный Markdown оказался полезен как материал для повторения: он дополняет текст книги, объясняет цель упражнения и основные конструкции, а исходный код остаётся доступен для чтения.

## Цель

Преобразовать каталог `audiobook` в простой учебный notebook с поясняющими конспектами по упражнениям книги.

Notebook должен:

- дополнять книгу, а не заменять её;
- помогать повторять материал после чтения;
- сохранять исходный код рядом с объяснением;
- иметь понятное оглавление;
- не содержать компонентов синтеза речи.

## Итоговая структура

```text
notebook/
├── README.md
└── part_02_quick_tasks/
    └── 04_argparse.md
```

Дополнительный уровень `text/` не нужен: всё содержимое notebook является текстовым учебным материалом.

## Содержимое конспекта

Файл `04_argparse.md` переносится без потери полезного материала. Формат текущего пилота становится основой для следующих конспектов:

1. название и ссылка на упражнение или страницы книги;
2. зачем нужно упражнение;
3. задача своими словами;
4. объяснение работы решения;
5. исходный код;
6. разбор важных конструкций;
7. вопросы для самостоятельной проверки.

Конспекты не должны копировать всю главу книги. Они кратко пересказывают учебную цель и добавляют объяснения, полезные начинающему разработчику.

## Оглавление

`notebook/README.md` объясняет назначение каталога и содержит ссылки на готовые конспекты. Первая запись ведёт к `part_02_quick_tasks/04_argparse.md` и указывает исходные страницы 42–44.

Отдельный файл прогресса внутри notebook не создаётся. Общий учебный прогресс уже ведётся в `materials/mentor_plan.md` и остаётся единственным источником состояния обучения.

## Удаляемые компоненты

После переноса текста удаляются компоненты отменённого аудиопилота:

- `audiobook/scripts/`;
- `audiobook/tests/`;
- `audiobook/pronunciation.json`;
- `audiobook/README.md`;
- `audiobook/progress.md`;
- правила `audiobook/output/` и `audiobook/tmp/` из `.gitignore`.

Каталог `audiobook` после миграции не остаётся.

## Исторические документы

Спецификация и план аудиопилота сохраняются в `docs/superpowers/` как история принятого решения. В спецификацию аудиопилота добавляется короткая пометка о том, что пилот завершён без одобрения голоса и заменён учебным notebook.

История Git с реализацией и отменой аудиопилота также сохраняется.

## Проверка результата

Миграция считается успешной, если:

- `notebook/README.md` ссылается на существующий конспект;
- `notebook/part_02_quick_tasks/04_argparse.md` содержит объяснения и полный листинг из `ext_04_argparse/ex04_argparse.py`;
- в рабочем дереве отсутствует каталог `audiobook`;
- по активным файлам проекта нет ссылок на команды синтеза речи;
- `materials/mentor_plan.md` остаётся единственным файлом прогресса;
- `git diff --check` не сообщает ошибок.

## Вне области работ

- создание конспектов для следующих упражнений;
- изменение решения `ex04_argparse.py`;
- генерация или хранение аудио;
- удаление FFmpeg из операционной системы;
- переписывание исторического плана аудиопилота.
