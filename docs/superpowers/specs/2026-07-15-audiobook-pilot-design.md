# Дизайн пилотной аудиоверсии книги
> **Статус:** пилот завершён без одобрения синтезированного голоса. Аудионаправление отменено; полезный текстовый материал перенесён в учебный `notebook`.

Дата: 2026-07-15

## Цель

Подготовить учебную аудиоверсию упражнения 4 про аргументы командной строки. Пилот должен подтвердить, что голос, темп, произношение терминов и способ объяснения кода подходят для дальнейшей обработки книги.

Полная книга не входит в пилот. Остальные упражнения обрабатываются только после прослушивания и одобрения пилотной дорожки пользователем.

## Принятые решения

- Навигация комбинированная: папки соответствуют частям книги, отдельные файлы - упражнениям.
- Пилотное упражнение: упражнение 4, аргументы командной строки и `argparse`.
- Голос: `Microsoft Irina` (`ru-RU`).
- Скорость: `Rate 0` для пилота.
- Формат: MP3, моно, 96 кбит/с.
- Листинги кода не читаются посимвольно. Перед каждым листингом даётся короткое учебное объяснение.
- Исходный код остаётся в сопроводительном Markdown.
- MP3 и временные WAV хранятся только локально и не попадают в Git.
- В Git сохраняются сценарии, адаптированные Markdown-файлы, словарь произношения, оглавление и прогресс.

## Структура файлов

```text
audiobook/
├── README.md
├── progress.md
├── pronunciation.json
├── scripts/
│   └── render_audio.ps1
├── text/
│   └── part_02_quick_tasks/
│       └── 04_argparse.md
├── output/
│   └── part_02_quick_tasks/
│       └── 04_argparse.mp3
└── tmp/
```

`audiobook/output/` и `audiobook/tmp/` добавляются в `.gitignore`.

## Поток данных

1. Текст упражнения извлекается из локального PDF или `materials/book_raw.md`.
2. Текст вручную сверяется с PDF и очищается от колонтитулов и ошибок распознавания.
3. В `04_argparse.md` сохраняются адаптированный текст, объяснения и исходные листинги кода.
4. Сценарий удаляет из копии для озвучивания Markdown-разметку и fenced-блоки кода.
5. Словарь произношения заменяет технические термины на варианты, понятные голосу Irina.
6. Microsoft Speech API создаёт временный WAV.
7. FFmpeg преобразует WAV во временный MP3: один канал, 96 кбит/с.
8. После технической проверки MP3 перемещается в `audiobook/output/`, а WAV удаляется.

## Формат учебного Markdown

Файл упражнения содержит:

- название и номер упражнения;
- номер части и страницы источника;
- очищенный основной текст;
- краткое объяснение перед каждым листингом;
- исходные листинги в fenced-блоках;
- команды и имена библиотек в исходном написании.

Сценарий читает обычный текст и объяснения, но пропускает:

- заголовочную метаинформацию;
- fenced-блоки кода;
- Markdown-разметку;
- длинные URL и служебные данные.

Сомнительный фрагмент OCR помечается маркером `[OCR-CHECK]`. При наличии такого маркера генерация останавливается, чтобы неподтверждённый текст не попал в аудио.

## Произношение

`pronunciation.json` хранит явные замены для TTS. Исходный Markdown при этом сохраняет нормальное написание терминов. Для пилота словарь включает как минимум `Python`, `argparse`, названия аргументов командной строки и часто встречающиеся английские команды.

После прослушивания пилота пользователь может один раз скорректировать скорость и словарь перед переходом к остальным упражнениям.

## Генерация и обработка ошибок

Перед генерацией `render_audio.ps1` проверяет:

- наличие входного Markdown;
- отсутствие `[OCR-CHECK]`;
- наличие текста после удаления разметки и кода;
- наличие голоса `Microsoft Irina`;
- доступность `ffmpeg` и `ffprobe`.

Готовый MP3 создаётся атомарно: сценарий сначала пишет временный файл и заменяет итоговый MP3 только после успешной проверки. При ошибке существующий готовый файл не изменяется. Временные файлы удаляются после успеха и сохраняются при ошибке для диагностики.

Сценарий завершает работу с ненулевым кодом и понятным сообщением, если отсутствует зависимость, найден непроверенный OCR, синтез речи не удался или FFmpeg вернул ошибку.

## Проверка пилота

Техническая проверка через `ffprobe` подтверждает:

- формат MP3;
- один звуковой канал;
- битрейт около 96 кбит/с;
- ненулевую длительность.

Проверка содержания включает:

- соответствие Markdown упражнению 4;
- отсутствие чтения листингов посимвольно;
- наличие понятных объяснений перед листингами;
- прослушивание начала, одного объяснения кода и окончания дорожки;
- отсутствие обрывов и явно неверного произношения.

## Критерий завершения

Пилот завершён, когда создан проверенный `04_argparse.mp3`, пользователь прослушал его и подтвердил голос, скорость и произношение. До этого подтверждения пакетная обработка остальных упражнений не выполняется.

## Зависимости и хранение

Голос Microsoft Irina уже доступен в Windows. FFmpeg отсутствует и устанавливается перед реализацией после системного разрешения.

Аудиофайлы предназначены для локального личного использования. В удалённый Git-репозиторий отправляются только текстовые данные и сценарии. `git push` выполняется только по явному запросу пользователя.
